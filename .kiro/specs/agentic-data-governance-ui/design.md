# UI/UX Design Document

## Overview

This document provides the detailed design specifications for the Agentic AI Data Governance Operating Model user interface. It defines the visual design system, component library, layout patterns, and interaction specifications that implement the requirements and workflows defined in the companion documents.

## Design System

### Color Palette

#### Primary Colors
- **Primary Blue**: #1E40AF (buttons, links, active states)
- **Primary Blue Light**: #3B82F6 (hover states)
- **Primary Blue Dark**: #1E3A8A (pressed states)

#### Semantic Colors
- **Success Green**: #059669 (pass indicators, positive trends)
- **Warning Amber**: #D97706 (warnings, pending states, data gaps)
- **Error Red**: #DC2626 (failures, critical issues, overdue)
- **Info Blue**: #0284C7 (informational messages)

#### Severity Colors
- **Critical**: #991B1B (dark red)
- **High**: #DC2626 (red)
- **Medium**: #D97706 (amber)
- **Low**: #0284C7 (blue)

#### Neutral Colors
- **Gray 900**: #111827 (primary text)
- **Gray 700**: #374151 (secondary text)
- **Gray 500**: #6B7280 (placeholder text)
- **Gray 300**: #D1D5DB (borders)
- **Gray 100**: #F3F4F6 (backgrounds)
- **White**: #FFFFFF (cards, panels)

#### AI Indicator Colors
- **AI Purple**: #7C3AED (AI-generated content indicator)
- **Human Blue**: #2563EB (human-verified content indicator)

### Typography

#### Font Family
- **Primary**: Inter (sans-serif)
- **Monospace**: JetBrains Mono (code, expressions)

## Interaction Patterns

### Drag and Drop
- **Kanban cards**: Drag between columns to change status
- **Threshold slider**: Drag to adjust DQ rule thresholds
- **Workflow steps**: Drag to reorder in configuration

### Inline Editing
- Double-click cell to edit in data grids
- Click pencil icon for explicit edit mode
- Escape to cancel, Enter to save
- Visual indicator for unsaved changes

### Bulk Actions
- Checkbox selection in data grids
- Floating action bar appears when items selected
- Actions: Approve, Reject, Assign, Export, Delete

### Progressive Loading
- Skeleton screens during initial load
- Infinite scroll for long lists
- "Load more" button as fallback
- Loading spinners for async operations

### Keyboard Shortcuts
| Shortcut | Action |
|----------|--------|
| Cmd/Ctrl + K | Global search |
| Cmd/Ctrl + S | Save current form |
| Escape | Close modal/drawer |
| Tab | Navigate form fields |
| Enter | Submit form / Confirm action |
| Arrow keys | Navigate lists |

## Accessibility Specifications

### Focus Management
- Visible focus ring (2px blue outline)
- Logical tab order following visual layout
- Focus trapped in modals
- Focus returned to trigger on modal close

### Screen Reader Support
- All images have alt text
- Form inputs have associated labels
- Status changes announced via aria-live
- Landmarks for main regions (nav, main, aside)

### Color and Contrast
- All text meets 4.5:1 contrast ratio
- Status indicators use icons + color
- Charts include patterns for colorblind users
- High contrast mode support

### Motion
- Respects prefers-reduced-motion
- No auto-playing animations
- Transitions under 200ms
- No flashing content

## Responsive Behavior

### Desktop (â‰¥1024px)
- Full sidebar navigation visible
- Multi-column layouts
- Side-by-side comparisons
- Full data grids

### Tablet (768px - 1023px)
- Collapsible sidebar (icon-only default)
- Stacked card layouts
- Simplified data grids (fewer columns)
- Drawers become full-screen modals

### Mobile (<768px)
- Bottom navigation bar
- Single column layouts
- Card-based data display
- Full-screen forms and modals
- Swipe gestures for navigation

## Animation Specifications

### Transitions
| Element | Property | Duration | Easing |
|---------|----------|----------|--------|
| Button hover | background-color | 150ms | ease-out |
| Drawer open | transform | 300ms | ease-out |
| Modal open | opacity, transform | 200ms | ease-out |
| Toast appear | transform | 200ms | ease-out |
| Dropdown open | opacity, transform | 150ms | ease-out |

### Micro-interactions
- Checkbox: Scale bounce on check (100ms)
- Button click: Scale down 2% (50ms)
- Card hover: Subtle shadow increase (150ms)
- Status change: Color fade (200ms)
